Games101中作业2为什么是倒着的？

作业2的正确目标结果

作业框架中参数 zNear、zFar 为正数。带入课堂视频中的投影矩阵可得出的结果

比较两个结果，不光是倒着的。X轴也是翻转的，所以对比下来，这两幅图片共有2处不同：

  1. X轴结果翻转
  1. Y轴结果翻转

为什么会有这样的结果？

正是因为改变了zNear 和 zFar的值。 由于 zNear 和 zFar的符号被改变。 投影矩阵中所有涉及到 这两个变量的参数的位置都会受到相应的影响。

所有参数 统计下来，X轴符号被改变1次（被翻转）。Y轴符号被改变1次（被翻转）。Z轴被改变3次（也同样被翻转）。改变符号的具体位置如下图标注 ↓

为什么要翻转z轴？

作业给的2个三角形。 第1个三角形 绿色的 z值为-2，第2个三角形 蓝色的 z值为-5. 相机的位置为5.

由此可以看出 绿色的三角形离相机更近。 所以渲染出的画面。 绿色三角形应该显示在前面。

作业描述中提到：

请注意我们是如何初始化 depth buffer 和注意 z values 的符号。为了方便 同学们写代码，我们将 z 进行了反转，保证都是正数，并且越大表示离视点越远。

那么作业框架中是如何初始化 depth buffer 的呢？

std::fill(depth_buf.begin(), depth_buf.end(),std::numeric_limits<float>::infinity());

从代码中可以看到 depth buffer 的初始值为最大。如果像素的 z值大于 depth buffer 才渲染。 那任何像素都不会比初始的 infinity 更大。所以此处的算法是 像素 z值 小于 depth buffer 才渲染。 也就是 z值越小越靠前。 那么再来看给定的两个三角形的 z values 的符号。 蓝色三角形z值为 -5 比 绿色三角形z值 -2 要更小 所以会被渲染到前面。 与 <绿色三角形离相机近，应该更靠前> 的这个结论相反。 所以要翻转像素点的z值。让更靠前的绿色三角形如期渲染到前面。

还有两处细节。

细节1：

// rasterizer.cpp 行86 float f1 = (50 - 0.1) / 2.0; float f2 = (50 + 0.1) / 2.0; // rasterizer.cpp 行108 vert.z() = vert.z() * f1 + f2;

此处没有改变 z值的符号

f1 是正数。 乘以一个正数没有改变 z值的符号。 f2也是正数。 加上1个正数 也不会改变z值的符号。 所以仅仅只是将z值的范围从标准立方体的 [1,-1]改变为[50,0.1]

细节2：

// rasterizer.cpp 行244 int rst::rasterizer::get_index(int x, int y) { return (height - 1 - y) * width + x; }

此处已经将 opencv 的屏幕坐标系的 y轴正方向向下， 翻转为 跟视频课程所用的右手坐标系一致。 所以结果不会受到opencv的影响而翻转画面的Y轴。
